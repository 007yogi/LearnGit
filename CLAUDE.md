## High‑Level Architecture & Structure

```
LearnGit/
├─ Pages/          ← HTML entry point (`index.html`).
├─ css/            ← Global stylesheet (`styles.css`).
├─ js/             ← Application logic (`script.js`).
└─ .git/           ← Git metadata (not part of the runtime).
```

- **HTML (`LearnGit/Pages/index.html`)** loads the CSS and the JavaScript bundle. It defines the page layout, navigation, the slider UI, and the care‑guide section.
- **CSS (`LearnGit/css/styles.css`)** contains all visual styling, custom properties (e.g., `--accent-color`), and the glass‑panel effects used throughout the UI.
- **JavaScript (`LearnGit/js/script.js`)** drives the interactive slider:
  * Holds a static `flowerData` array describing each flower (name, botanical name, rarity, description, care specs, accent colors, etc.).
  * Manages DOM references, navigation (buttons, keyboard, touch), autoplay with a progress bar, and updates the details panel when the active slide changes.
  * Uses CSS custom properties to dynamically change accent colors and ambient glow for the currently selected flower.
- **No build tooling** – the site is a classic static HTML/CSS/JS stack. All assets are referenced via relative URLs, so the site can be served directly from the filesystem or any static‑file web server.

### Interaction Flow
1. **Page Load** – `DOMContentLoaded` triggers `initSlider()`.
2. `initSlider()` registers click, keyboard, and touch listeners, sets the first slide active, and starts the autoplay timer.
3. When a slide changes (`goToSlide` / `navigateSlider`):
   - UI classes are updated (`active` slide, dot indicator).
   - CSS variables `--accent-color` and `--accent-glow` are set from the selected flower data.
   - The ambient glow element receives a radial gradient based on `glow`.
   - The details panel fades out, updates its text content, then fades back in.
   - Autoplay timer resets, and the progress bar visualizes the remaining time until the next slide.
