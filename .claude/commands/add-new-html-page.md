# Command: `Add-New-HtmlPage`

> **Type:** Custom Command  
> **Trigger:** `/Add-New-HtmlPage`  
> **Purpose:** Generates a new, fully linked HTML page and integrates it into the project navbar automatically.

---

## What This Command Does

When invoked, this command will:

1: Before creating the page ask for the permission.
2: Ask The html page Name that i want to create.
2. Ask for basic page details (name, filename, CSS path, JS path, existing navbar)
3. Generate a complete, ready-to-use `.html` file


### Step 1 — Gather Page Information

Before generating any file, collect the following details from the user.  
If any value is missing, use the **default** shown below.

| # | Information Needed        | Example Value            | Default              |
|---|---------------------------|--------------------------|----------------------|
| 1 | Page title / name         | `About`, `Contact`       | `New Page`           |
| 2 | HTML filename             | `about.html`             | `new-page.html`      |
| 3 | CSS file path             | `css/styles.css`         | `css/styles.css`     |
| 4 | JS file path              | `js/main.js`             | `js/main.js`         |
| 5 | Existing `<nav>` HTML     | *(paste from project)*   | Basic `<nav>` stub   |
---

### Step 2 — Generate the New HTML Page

Create a new file named `{PAGE_FILENAME}` using the structure below.  
Replace every `{PLACEHOLDER}` with the real values.

**Placeholder reference:**

| Placeholder       | Replace With                        | Example              |
|-------------------|-------------------------------------|----------------------|
| `{PAGE_TITLE}`    | Full page title for `<title>` tag   | `About Us`           |
| `{PAGE_FILENAME}` | The `.html` filename                | `about.html`         |
| `{PAGE_NAME}`     | Short display name for nav link     | `About`              |
| `{CSS_PATH}`      | Relative path to CSS file           | `css/styles.css`     |
| `{JS_PATH}`       | Relative path to JS file            | `js/main.js`         |

---

### Step 3 — Update the Navbar on All Other Existing Pages

The new page link must be added to every other `.html` file in the project so the navbar stays consistent across all pages.

**Add this `<li>` to the `<nav>` on every other page:**

```html
<li><a href="{PAGE_FILENAME}">{PAGE_NAME}</a></li>
```

**Important rules when updating other pages:**

- Do **not** add `class="active"` on other pages — only the current page gets `active`
- Keep the **same link order** across all pages
- Do **not** change any other part of the existing navbar
- Do **not** changes in previously created pages.
- Add Header And Footer in top and bottom.

## File & Folder Structure Reference

This command assumes a standard flat or one-level project structure:

```
project/
│
├── index.html            ← existing home page
├── about.html            ← example existing page
├── contact.html          ← new page (generated by this command)
│
├── css/
│   └── styles.css        ← shared stylesheet (linked in <head>)
│
└── js/
    └── main.js           ← shared script (linked before </body>)
```

## Rules & Best Practices

| Rule | Details |
|------|---------|
| ✅ CSS in `<head>` | Always link stylesheets inside `<head>`, never in `<body>` |
| ✅ JS with `defer` | Use `<script src="..." defer></script>` so JS runs after HTML parses |
| ✅ Active class | Add `class="active"` only on the nav link matching the current page |
| ✅ Relative paths | Use relative paths (`css/styles.css`) not absolute (`/css/styles.css`) unless the project requires it |
| ✅ Copy existing nav | Never rewrite the user's navbar — copy it and insert the new link only |
| ✅ Match code style | Use the same indentation (spaces or tabs) as the rest of the project |
| ❌ No duplicate navbars | Do not create a second `<nav>` block |
| ❌ No inline styles | Do not add `style=""` attributes; use the linked CSS file |
| ❌ No inline scripts | Do not use `<script>` blocks in the HTML body; use the linked JS file |
